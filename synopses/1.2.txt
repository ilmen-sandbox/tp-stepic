Internet и WWW
------------------------------------------------------------------------------

Internet - глобальная сеть передачи данных
  Протоколы образуют стек (когда одни протоколы работают поверх других)
  Стек протоколов Internet (избранные):
    Протокол IP
      позволяет передавать данные между двумя машинами сети
      вводит глобальную адресацию устройств внутри сети
      работает с datagram (кусочками данных)
      не гарантирует надежную доставку данных (UDP?)
    Протокол TCP (поверх IP)
      строго гарантированная доставка
      время доставки не гарантируется
      данные если протокол отвечает что данные доставлены, то это точно так
    Протоколы HTTP, FTP, P2P, SSH (поверх TCP)

WWWW
  1) WWW - множество гипертекстовых (взаимосвязанных) документов интернета (подмножество internet)
  2) WWW - множество протоколов передачи гипертекстовых документов
  Гиперссылка - способ указания связи двух документов
  
MIME - типы
------------------------------------------------------------------------------
  Возникли для передачи файлов в виде e-mail, где все представленно в виде теста.
  Именно MIME-тип определяет как и чем обрабатывать файл (не расширение файла, а именно MIME-тип).
  
  XML (text/xml) - это более строгий (в сравнении с HTML) формат описания документа
    Допускает разные форматы записи (поля объекта - вложенные теги объекта или поля объекта - атрибуты тега объекта)
  JSON (application/json) - совокупность массивов ([ "значение" ]) и словарей ({ "ключ" : "значение" }) на языке JS
    Позволяет легко представить дерево в виде тествого файла.
    
  Документы
    Статические - как правило, файл
    Динамические - генерируется на лету в application server по запросу пользователя
    
    
URL (URI)
------------------------------------------------------------------------------
  URL - подмножество URI
  URI - Unified Resource Identificator
  URL - Unified Resource Locator
  
  Если у документа нет URL, то его невозможно получить из WWW.
  
  Состав абсолютного URL:
    http://server.org:8080/path/doc.html?a=1&b=2#part1
      http - протокол
      :// - разделитель
      server.org - DNS или IP
      8080 - TCP порт
      /path/doc.html - путь к файлу
      a=1&b=2 - QUERY-string
      part1 - якорь, позиция в документе, откуда начать просмотр
  Schemeless
    //server.org/1.html
    URL без указания протокола для абстрагирования от использования HTTP И HTTPS
    Если документ отдается по HTTPS, то все вложенные документы должны передаваться по HTTPS
  Отновительные URL
    /another/page.html?a=1 - относительный (в пределах домена, относительно протокола и домена текущего сайта)
    pictures/1.png - относительный (от URL текущего документа)
    ?a=1&b=2 - относительный (от URL текущего документа)
  Якорь, относительный URL в текущем документе (без перегрузки)
    #part2

  Правила разрешения URL
    Пусть дан основной документ: https://site.com/path/page.html
    Тогда URL строятся так:
      http://wikipedia.org  = http://wikipedia.org
      //cdn.org/jquery.js   = https://cdn.org/jquery.js (так как основной документ получен по HTTPS)
      /admin/index.html     = https://site.com/admin/index.html
      another.html          = https://site.com/path/another.html
      ?full=1               = https://site.com/path/page.html?full=1    (если у основного документа были QUERY-string параметры, то они будут отброшены)
      #chapter2             = https://site.com/path/page.html#chaprer2  (не приводит к перезагрузке страницы)
  
Способы перехода от одного документа на другой
  - С помощью гиперссылки <a href=.../>
  - С помощью передачи формы <form>...<form/>
  - Метаданные <link..../>, <script src=...>...<script/>
  - Спец. теги <img src=.../>
  - Фреймы и т.п.
  
Ссылки вв CSS ресурсах
  Картинки, шрифты и другое можно указать как ресурс в CSS файле.
  .slide {
    background-image: url(../pictures/network.png)
  }
